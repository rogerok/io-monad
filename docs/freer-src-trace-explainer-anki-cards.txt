Что хранит `FreerImpure` в текущем `src/freer`?	`op`, то есть один запрос эффекта, и `cont`, функцию “ответ эффекта -> следующая Freer-программа”.
Почему `freerBind` не делает switch по `readLine`, `writeLine`, `fetch`?	Потому что конкретный эффект лежит в `op`, а `bind` меняет только универсальное поле `cont`.
Что означает фантомное поле `_resp` в `Instr`?	Тип ответа инструкции для TypeScript; в runtime оно не несёт поведения.
Что делает `freerRun` после выполнения `world.readLine()`?	Передаёт полученную строку в `current.cont(value)` и продолжает цикл с возвращённым Freer-узлом.
Зачем `freerDo` оборачивает generator в `freerSuspend`?	Чтобы generator создавался при интерпретации, а не при сборке программы; так один `FreerIO` можно запускать многократно.
Что отдаёт `Symbol.iterator` в `freerMk`?	`YieldWrap(this)`, чтобы `yield* freerNode` мог вернуть тип результата Freer-узла.
Почему `yield* freerReadLine` не выполняет чтение?	`yield*` только отдаёт Freer-узел наружу в `freerDo`; чтение выполняет `freerRun` через `world.readLine`.
Какие места нужно изменить для нового эффекта в текущем Freer?	Тип инструкции в `Instr`, smart constructor и ветку interpreter-а в `freerRun`.
Почему `freerRandom` тестируем без настоящей случайности?	Потому что interpreter берёт число из `FreerWorld.random`, который можно заменить в тесте.
Какая главная архитектурная выгода Freer по сравнению с монолитным IO?	`bind` отделён от конкретных инструкций, поэтому монадическое ядро не меняется при добавлении эффекта.
